## 共通前提(依頼文の冒頭に必ず含める)

```
【プロダクト共通前提】
中高生(13-18歳)向けの強み発見サービスの結果画面。
守るべき原則(優先順):
1. タイプ・成長方向に優劣・序列・レア度を持ち込まない。「上位」「進化」は禁止語彙
2. 断定形(〜です/〜になる)禁止。発見形(〜みたい/〜かも)で書く
3. 目標はタイプではなく「のばす力(コンピテンシー)」に置く。タイプ変化は結果として見つかる発見
4. グレーアウト=ロック・欠損の表現をしない。未到達は「この先で見えてくる」フレーム
5. 主語は「キミ」。1文40字以内。感嘆符は1画面1個まで
6. AIのおすすめは参考であり、選択の主体は常に生徒。根拠(診断結果)を必ず添える
不明点は推測で埋めず「要確認」と明示してください。
```

## 修正後に必須の検証(実装前チェック)

1. **12タイプパリティ検証**:全12タイプを中心に置いた場合のマップを横並びで確認。①「いける未来(隣接ハイライト)」の数が全タイプ同数か ②中心の切替で隣接関係に矛盾がないか(教室で隣の席と見比べる前提) ③おすすめバッジの出現頻度・分岐先の魅力にタイプ間格差がないか
2. **隣接ルールの文書化**:「なぜこのタイプの隣がこのタイプか」を診断モデル(コンピテンシー軸/Big5軸の距離)で説明できる定義書を1枚作成(7/22ゲートの仕様根拠を兼ねる)
3. **再診断時の再会コピー**:チャレンジ後の再診断で目標方向のタイプに**ならなかった場合**の肯定コピーを設計(「チャレンジ失敗」体験の構造的防止)。※本画面の外だが本施策とセットで必須

## 要人間判断(AIに決めさせない)

- #1 対語彙(みがく/ひろげる 等)の最終決定 → 教育BUと合意
- #3 「統率者(リーダー)」の代替名
- おすすめアルゴリズムの根拠開示レベル(教師レイヤーへの可視化仕様含む)
